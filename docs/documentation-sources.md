# Карта документации и источников контекста

## Назначение

Этот файл объясняет, какие документы в репозитории являются каноническими, какие поддерживаются как operational-документы, а какие нужны только для навигации.

## Канонические источники

- `README.md`
  - главный обзор проекта, запуск, структура, ограничения и release-flow.
- `.github/ai-context.md`
  - рабочий контекст для AI-агентов и разработчиков, правила и ограничения.
- `docs/improvements.md`
  - backlog улучшений, статус задач и приоритеты.
- `src/**`
  - фактический источник истины при споре между кодом и документацией.

## Поддерживаемые operational-документы

- `docs/deploy_layout.md`
  - схема размещения файлов в deploy-контуре и runtime-пути.
- `docs/release_notes.md`
  - журнал релизных изменений и важных действий после релиза.
- `.github/ai-agent-technical-task.md`
  - конкретное ТЗ для AI-агента; читать только по прямому запросу пользователя.

## Вторичные обзорные документы

- `docs/repository-root-map.md`
  - карта верхнего уровня репозитория.
- этот файл
  - карта ролей документов и рекомендуемого порядка чтения.

## Рекомендуемый порядок чтения

1. `README.md`
2. `.github/ai-context.md`
3. `docs/improvements.md`
4. нужные файлы из `src/`
5. `docs/deploy_layout.md` при задачах по deploy
6. `docs/release_notes.md` при задачах по релизу или анализу изменений
7. `.github/ai-agent-technical-task.md` только если пользователь прямо попросил выполнять конкретное ТЗ

## Что не считать главным источником правды

- обзорные карты в `docs/` (`docs/repository-root-map.md`, этот файл)
- `test/` и `tests/`, кроме согласованных smoke-скриптов handlers
- `venv/`, `.env`, `__pycache__/`
- устаревшие локальные материалы в `local/`, если они не подтверждены кодом
