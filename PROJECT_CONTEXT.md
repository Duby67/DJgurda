# PROJECT_CONTEXT.md

## 🤖 КОНТЕКСТ ДЛЯ ИИ-АССИСТЕНТА

**Цель этого файла:** Быстро передать полный контекст проекта при запросе помощи. Содержит детальную информацию об архитектуре, текущем состоянии и технических решениях.

## 🎯 Общее описание проекта
*   **Название бота:** DJgurda Bot
*   **Основная цель:** Telegram бот для обработки медиа-ссылок из групповых чатов. Преобразует ссылки на контент из различных платформ (TikTok, YouTube, Instagram, Яндекс.Музыка) в унифицированный формат, доступный всем участникам беседы.
*   **Целевая аудитория:** Пользователи Telegram групповых чатов, где участники имеют доступ к разным медиа-платформам.

## 🏗️ АРХИТЕКТУРА ПРОЕКТА

### 🛠 Технологический стек
- **Язык:** Python 3.11
- **Фреймворк:** aiogram 3.x (асинхронный)
- **База данных:** SQLite + SQLAlchemy + aiosqlite
- **Медиа обработка:** yt-dlp, FFmpeg
- **Контейнеризация:** Docker + Docker Compose
- **CI/CD:** GitHub Actions → автоматический деплой на VPS
- **Среда выполнения:** VPS с Docker (две среды: dev и prod)

### Ключевые ограничения
- Максимальные размеры файлов: фото 10MB, видео/аудио 50MB
- Длина подписей: до 1024 символов
- Параллельные загрузки: Semaphore(3)
- Время жизни временных файлов: 3600 секунд

## 🚀 Основная функциональность

**Ядро системы:**
- Полная асинхронная архитектура на aiogram 3.x
- Production-ready Docker контейнер с FFmpeg
- Автоматический CI/CD пайплайн с деплоем на VPS
- Раздельные окружения dev/prod с изоляцией

**Обработка медиа:**
- TikTok - полная реализация (video/photo/profile/слайдшоу)
- YouTube - базовая структура обработчика
- Instagram - базовая структура обработчика  
- Яндекс.Музыка - базовая структура обработчика
- Интеллектуальное определение типа контента по URL
- Поддержка групп медиа (слайдшоу с аудио)

**Система управления:**
- Команды: `/help`, `/info`, `/statistics`, `/toggle_bot`, etc.
- Middleware для управления доступностью бота в чатах
- Статистика использования с SQLite базой данных
- Система эмодзи с кастомными Telegram Emoji ID

**Безопасность и надежность:**
- Проверка размеров файлов (photo: 10MB, video/audio: 50MB)
- Ограничение параллельных загрузок (Semaphore)
- Автоматическая очистка временных файлов
- Обработка ошибок с резервными стратегиями

**Инфраструктура:**
- FFmpeg предустановлен для обработки медиа
- Автоматическое создание директорий для временных файлов
- Volume для хранения базы данных
- Переменные окружения через .env файлы

**Реализованные обработчики:**
- ✅ TikTok - полная поддержка видео, фото-слайдшоу и профилей
- ◻️ YouTube - заглушка (требует реализации)
- ◻️ Instagram - заглушка (требует реализации)

## 💾 Архитектура обработчиков

**Базовые компоненты:**
- `BaseHandler` - абстрактный класс для всех обработчиков
- `ServiceManager` - регистрация и поиск обработчиков по URL
- Миксины (`VideoMixin`, `PhotoMixin`, `AudioMixin`, `MediaGroupMixin`) - общая функциональность

**Поддерживаемые типы контента:**
- Видео (с миниатюрами)
- Аудио (с обложками)
- Фото и группы медиа (слайдшоу)
- Профили пользователей

## 💾 Система команд бота

**Основные команды:**
- `/help` - список поддерживаемых медиа-платформ
- `/info` - информация о версии и времени работы бота
- `/statistics` - статистика активности в чате (топ пользователей)

**Команды управления:**
- `/start`, `/stop`, `/toggle_bot` - включение/отключение бота в чате
- `/enable_errors`, `/disable_errors`, `/toggle_errors` - управление показом ошибок
- `/enable_notifications`, `/disable_notifications`, `/toggle_notifications` - управление уведомлениями

## 💾 Данные и база данных

**Основные сущности:**
- Настройки бота (bot_settings) - состояние бота в чатах
- Статистика (stats) - активность пользователей
- Источники (sources) - поддерживаемые медиа-платформы

**Ограничения:**
- `MAX_CAPTION = 1024` - максимальная длина подписи
- `MAX_AGE_SECONDS = 3600` - время жизни кэша
- Лимиты размеров файлов: фото 10MB, видео/аудио 50MB

## 🔧 Конфигурация и переменные окружения

**Обязательные переменные:**
- `BOT_TOKEN` - токен Telegram бота
- `BOT_DB_PATH` - путь к базе данных
- `YANDEX_MUSIC_TOKEN` - токен Яндекс.Музыки
- `YOUTUBE_COOKIES_PATH` - путь к cookies для YouTube
- `ADMIN_ID` - ID администратора бота
- `BOT_VERSION` - версия бота (генерируется автоматически)
